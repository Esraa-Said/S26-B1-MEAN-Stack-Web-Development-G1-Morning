🎯 **Task Title: Enhance Your Module with Real Backend Features**

---

## ✅ Objective:

In this task, you will upgrade the first module you already built in your graduation project by adding professional backend features used in real-world APIs:

* Filtering data
* Pagination
* Sorting
* File upload using Multer

---

## 📌 Requirements:

---

## 🧩 1. Use Your Existing Entity

Continue working on the same entity you created in the previous task (Doctor, Product, Course, Book, etc.).


---

## 🔍 2. Filtering Feature

Allow users to filter data using query parameters.

### What you should support:

* Filtering by fields like category, name, price, etc.
* Supporting advanced filtering operators such as:

  * greater than
  * less than
  * greater than or equal
  * less than or equal

---

## 📄 3. Pagination Feature

Your API should support splitting data into pages.

### Requirements:

* Users can control:

  * page number
  * number of items per page
* Your response should include:

  * current page
  * items per page
  * total number of items
  * total number of pages

---

## ↕️ 4. Sorting Feature

Allow users to sort results based on any field.

### Requirements:

* Support ascending sorting
* Support descending sorting
* Default sorting should return newest data first

---

## 📁 5. File Upload Feature (Multer)

Add file upload functionality to your module.

### Requirements:

* Upload a file (preferably an image) related to your entity
* Save the uploaded file with the entity data in the database
* Ensure the upload works properly with your API requests

---

## 🧪 6. Testing Requirements (Postman):

You must test all features using Postman:

* Create new item with file upload
* Retrieve all items with:

  * filtering
  * pagination
  * sorting
* Get single item by ID
* Update item
* Delete item

📸 Take screenshots for each successful test.

---

## 🧾 7. Submission:

* Push your updated project to GitHub
* Keep everything organized inside your existing project
* Update your README file to include:

  * Features you implemented
  * How to run the project
  * Examples of API usage (described, not coded)
